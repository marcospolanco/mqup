# MQUP — Maps Query Understanding Prototype

On-device iOS prototype for compound natural-language POI search. Combines **SQLite FTS5 (BM25)** with **deterministic hash embeddings** (384-dim stand-in; Core ML MiniLM staged — see `models/README.md`), ranked through a constraint-aware hybrid blend and exposed via **SwiftUI**, **App Intents**, and **Spotlight**.

## Architecture

```
SearchCoordinator.submit(query:)
  → QueryIntentExtractor
  → HybridRanker (hard filter → BM25 + embeddings → blend → soft demotion)
  → SearchResultsBuilder.buildSearchResultsView
  → SearchResultsScreen (SwiftUI + MapKit)
```

Siri uses the same coordinator and `buildSiriDialog(from:)` — no ad-hoc string formatting in `SearchPlacesIntent.perform()`.

## Quick start

```bash
python3 scripts/generate_pois.py
python3 scripts/generate_eval_queries.py   # writes eval/query_templates.json
python3 scripts/generate_fixtures.py
swift test
swift build -c release
.build/release/MQUPEval --label          # constraint + token-overlap labels (frozen; independent of hybrid)
.build/release/MQUPEval                  # tune α on dev, evaluate against frozen labels
```

## Evaluation

Metrics come from `metrics.json` (generated by `MQUPEval` release build). Re-run eval before citing numbers in an interview.

| Metric | Target | Actual (honest labels) | Pass |
|--------|--------|------------------------|------|
| nDCG@5 delta (hybrid − BM25) | +0.10 | **+0.01** | ✗ |
| Recall@10 | ≥0.85 | **0.12** | ✗ |
| p95 latency (hybrid, macOS) | <100ms | **5ms** | ✓ |

- **Tuned α:** 0.50 (120-query dev split)
- **Labels:** Constraint satisfaction + token-overlap ranking — **frozen before α tuning**, never derived from hybrid top-10 (see `ConstraintRelevanceLabeler`)
- **Blind hold-out:** Same labeling methodology; templates fixed before tuning
- **Resume bullet:** Auto-regenerated in `docs/resume-bullet.md` from the same run

Previous oracle labels (hybrid top-10 as relevance) inflated recall to ~1.0. Honest constraint labels expose ranking gaps — the expected outcome until Core ML embeddings and/or human relevance judgments land.

Latency is measured on the **macOS release harness**, not iPhone hardware. Re-profile on iPhone 14+ before citing device latency.
